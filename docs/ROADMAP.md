# 本译开发路线图

路线图按交付能力划分，不承诺固定发布日期。每个阶段只有在对应验收条件满足后才进入下一阶段。

## M0：需求与技术验证

- [x] 明确产品定位、目标用户和非目标。
- [x] 定义首版功能需求和验收框架。
- [x] 定义扩展组件、消息协议和隐私边界。
- [ ] 在受支持 Chrome 上验证英语到简体中文的语言检测、可用状态和首次准备流程。
- [x] 将长期翻译迁移到隐藏文档，并验证控制面板关闭后任务不暂停。
- [ ] 验证输入配额、语义分块、AbortSignal 和迟到结果丢弃。
- [ ] 使用固定长文页面测量首个可见段落延迟 P50/P95、队列、长任务和内存表现，并冻结发布阈值。
- [ ] 选定 SPA 导航失效方案并确认所需权限。
- [ ] 通过网络检查确认模型下载之外没有正文或译文离开设备。

退出条件：所有 P0 技术风险均有可行验证结果或明确降级方案。

## M1：扩展工程骨架

- [x] 创建 Manifest V3 工程、构建和测试配置。
- [ ] 增加自动格式化和代码风格检查。
- [x] 实现 Service Worker、Content Script 和 Side Panel 的基本通信，以及 `tabId + pageId + taskId` 身份校验。
- [ ] 实现设置存储、版本迁移和诊断状态。
- [x] 建立固定网页夹具和自动化测试基线。
- [ ] 加入无障碍和安全静态检查。

退出条件：扩展可本地加载，组件通信稳定，CI 能运行全部基础检查。

## M2：整页双语翻译 MVP

- [ ] 实现文本发现、排除规则和稳定段落映射。
- [ ] 实现语言识别、可用性检查和首次下载进度。
- [ ] 实现视口优先队列、分块、取消和错误隔离。
- [ ] 实现双语渲染、三种显示模式和完整撤销。
- [ ] 实现动态内容监听与单页应用导航失效。
- [ ] 实现页面内去重和有容量上限的内存缓存。

退出条件：满足产品需求文档中的全部首版验收标准。

## M3：公开测试版本

- [x] 实现划词浮层、右键菜单和快捷键。
- [ ] 实现站点黑名单、自动翻译规则和可选站点权限。
- [ ] 实现默认关闭的持久化缓存、缓存管理界面和译文样式设置。
- [ ] 实现混合语言页面的文本单元级语言检测。
- [ ] 完成三大桌面平台兼容性矩阵。
- [ ] 完成隐私说明、商店截图、发布说明和支持流程。
- [ ] 根据测试用户反馈修复高优先级兼容问题。

退出条件：无阻塞级缺陷，隐私和权限说明通过发布检查。

## M4：增强能力

- [ ] 术语表与固定译法。
- [ ] 输入框双向翻译辅助。
- [ ] 视频字幕双语显示。
- [ ] 普通文本 PDF 支持。
- [ ] 本地摘要、解释和上下文润色。
- [ ] 更丰富的站点适配规则。

进入条件：核心翻译稳定，增强能力不会扩大默认数据边界。

## 暂不排期

- 图片和扫描 PDF OCR。
- 移动端浏览器支持。
- 云端账户、同步和远程翻译服务。
- 独立桌面端和其他浏览器实现。
