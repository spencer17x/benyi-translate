# 本译产品需求文档

## 1. 文档信息

| 字段 | 内容 |
| --- | --- |
| 产品名称 | 本译 Benyi |
| 产品形态 | Chrome Manifest V3 扩展 |
| 当前阶段 | 需求设计 |
| 核心定位 | 本地、隐私、沉浸式网页翻译 |
| 默认目标语言 | 简体中文 `zh` |

需求优先级定义：

- **P0**：首个可用版本必须完成。
- **P1**：公开测试版本应完成。
- **P2**：产品增强项，不阻塞首版。

## 2. 产品愿景

让用户在不提交网页内容到远程翻译服务的前提下，随时获得自然、连续、可撤销的双语阅读体验。

## 3. 目标用户

1. 经常阅读英文、日文、韩文等外语网页的中文用户。
2. 对网页内容隐私敏感，不希望将全文发送到第三方服务的用户。
3. 在弱网或离线环境下仍需要翻译能力的用户。
4. 需要保留原文进行对照阅读、语言学习或专业校对的用户。

## 4. 产品目标

### 4.1 首版目标

- 用户可以通过一次明确操作翻译当前网页。
- 翻译结果以原文和译文对照形式呈现。
- 当前视口内容优先翻译，页面可在翻译期间继续阅读和操作。
- 翻译过程不依赖远程推理服务，不要求用户配置 API Key。
- 用户可以暂停、取消、重试、撤销翻译并清除本地数据。

### 4.2 非目标

以下能力不属于首版范围：

- 移动端 Chrome 支持。
- 云端翻译服务或账户同步。
- 图片 OCR、扫描 PDF 和 Canvas 文字识别。
- 全功能视频字幕翻译。
- 替代专业人工翻译或提供法律、医疗等领域的准确性保证。
- 绕过受保护页面、浏览器内部页面或网站访问控制。

## 5. 核心用户旅程

### 5.1 首次使用

1. 用户安装扩展并打开引导页或侧边栏。
2. 产品检测浏览器、设备和本地语言能力是否可用。
3. 用户选择默认目标语言并点击“准备本地翻译”。
4. 如需下载语言资源，产品展示明确的下载状态和进度。
5. 准备完成后，产品提供一键测试和隐私说明。

### 5.2 翻译当前网页

1. 用户点击扩展按钮、侧边栏按钮或快捷键。
2. 产品识别页面主要语言并收集可阅读文本。
3. 产品优先翻译当前视口中的段落。
4. 译文插入原文下方，后续内容在滚动和空闲期间继续翻译。
5. 用户可以切换显示模式、暂停或撤销翻译。

### 5.3 翻译选中文本

1. 用户在网页中选择一段文本。
2. 产品显示轻量翻译入口或响应右键菜单命令。
3. 翻译结果显示在靠近选区的浮层中。
4. 用户可以复制译文、关闭浮层或打开更多解释。

## 6. 功能需求

### 6.1 环境检测与首次准备

| 编号 | 优先级 | 需求 |
| --- | --- | --- |
| ENV-01 | P0 | 扩展必须检测核心语言 API 是否存在，不能仅依赖浏览器版本号。 |
| ENV-02 | P0 | 扩展必须检测指定语言对的可用状态，并区分可用、需下载、下载中和不可用。 |
| ENV-03 | P0 | 首次下载必须由明确的用户操作触发，并显示进度、成功或失败状态。 |
| ENV-04 | P0 | 设备或语言对不可用时，必须提供可理解的原因和重试入口。 |
| ENV-05 | P1 | 设置页应提供诊断信息复制功能，且不得包含网页正文。 |

### 6.2 网页文本发现

| 编号 | 优先级 | 需求 |
| --- | --- | --- |
| DOM-01 | P0 | 提取用户可见且有阅读价值的文本节点，并按段落形成翻译单元。 |
| DOM-02 | P0 | 默认排除 `script`、`style`、`noscript`、表单控件、可编辑区域和扩展自身 UI。 |
| DOM-03 | P0 | 默认不翻译空白、纯标点、URL、过短噪声和已经是目标语言的内容。 |
| DOM-04 | P0 | 不得通过替换整段 `innerHTML` 的方式更新页面。 |
| DOM-05 | P0 | 每个翻译单元必须能稳定映射回原始节点，以支持撤销和动态更新。 |
| DOM-06 | P1 | 用户可选择是否翻译代码块、引用、导航、按钮和表格。 |
| DOM-07 | P1 | 对开放的 Shadow DOM 提供尽力支持。 |
| DOM-08 | P2 | 对获得访问权限的同源或可注入 iframe 提供翻译支持。 |

### 6.3 整页翻译

| 编号 | 优先级 | 需求 |
| --- | --- | --- |
| PAGE-01 | P0 | 用户可以一键开始翻译当前页面。 |
| PAGE-02 | P0 | 当前视口及其临近内容必须优先于页面其余内容进入翻译队列。 |
| PAGE-03 | P0 | 同一语言对的任务必须受控排队，避免无界并发。 |
| PAGE-04 | P0 | 用户可以暂停、继续和取消当前页面翻译。 |
| PAGE-05 | P0 | 取消后不得继续写入迟到的翻译结果。 |
| PAGE-06 | P0 | 用户可以恢复原始页面，扩展插入的译文和样式应被完整移除。 |
| PAGE-07 | P0 | 页面导航、刷新或单页应用路由变化后，旧任务必须失效。 |
| PAGE-08 | P1 | 用户可为站点开启“进入后自动翻译”。 |
| PAGE-09 | P1 | 支持按站点保存源语言、目标语言和显示模式。 |

### 6.4 动态内容

| 编号 | 优先级 | 需求 |
| --- | --- | --- |
| DYN-01 | P0 | 使用 DOM 变化监听发现新增的可阅读内容。 |
| DYN-02 | P0 | 对短时间内的大量变化进行合并和节流，避免重复扫描。 |
| DYN-03 | P0 | 已翻译且内容未变化的节点不得重复排队。 |
| DYN-04 | P1 | 对无限滚动页面，仅处理进入临近视口范围的新增内容。 |

### 6.5 翻译执行

| 编号 | 优先级 | 需求 |
| --- | --- | --- |
| TR-01 | P0 | 自动检测页面或文本单元的源语言，用户可以手动覆盖。 |
| TR-02 | P0 | 使用浏览器支持的本地语言对执行翻译。 |
| TR-03 | P0 | 翻译前必须规范化文本，但不得改变用户可见的原文。 |
| TR-04 | P0 | 对重复文本去重，同一页面内相同输入和语言对应复用结果。 |
| TR-05 | P0 | 超出单次输入配额的内容必须按句子或语义边界安全分块。 |
| TR-06 | P0 | 每个任务必须支持取消、超时和失败重试。 |
| TR-07 | P0 | 语言资源失效或被系统回收时，必须重新进入可用性和下载流程。 |
| TR-08 | P1 | 支持用户术语表，在展示前对符合规则的译文进行可控替换。 |
| TR-09 | P2 | 提供上下文润色、术语解释和双语摘要等增强能力。 |

## 7. 非功能需求

### 7.1 隐私

- P0：核心功能不得把网页正文、选中文本或译文发送到远程服务器。
- P0：默认不启用分析、遥测或用户行为追踪。
- P0：日志不得记录完整网页正文；调试日志中的文本必须截断或脱敏。
- P0：权限申请必须与功能对应，并在安装说明中解释用途。

### 7.2 性能

- P0：模型已可用时，用户触发后 500 毫秒内进入明确的“正在翻译”状态。
- P0：优先队列必须保证首屏内容先于屏外长文处理。
- P0：DOM 扫描、观察和渲染应分批执行，避免持续阻塞页面主线程。
- P0：同一文本不得重复推理，除非用户明确要求重新翻译。
- P1：设置页提供本次页面的段落数、缓存命中和失败数量，不记录正文。

### 7.3 可靠性

- P0：单个段落失败不得中断整页翻译。
- P0：页面关闭或导航后必须释放任务、观察器和消息连接。
- P0：扩展重新加载后，页面中残留的扩展节点应可识别并安全清理。
- P1：队列异常时可重建，不要求用户刷新网页。

### 7.4 可访问性

- P0：主要操作可通过键盘完成。
- P0：按钮、状态和进度应提供可读的无障碍名称。
- P0：浮层必须支持 Esc 关闭，并正确管理焦点。
- P1：支持减少动态效果和高对比度偏好。

### 7.5 安全

- P0：不得使用 `eval` 或执行来自网页内容的脚本。
- P0：译文必须使用文本节点或等价安全方式写入 DOM。
- P0：网页发送给扩展的消息一律视为不可信输入并校验结构。
- P0：发布包不得在运行时下载并执行远程 JavaScript。

## 8. 首版验收标准

首版必须同时满足以下条件：

1. 在受支持的 Chrome 桌面环境中，可以完成英语网页到简体中文的本地双语翻译。
2. 首次语言资源下载由用户操作触发，并有完整状态反馈。
3. 当前视口优先翻译，长页面支持暂停、继续、取消和撤销。
4. 动态新增内容能够进入翻译队列，且不会反复翻译同一节点。
5. 选中文本可通过右键菜单翻译和复制。
6. 翻译失败不会破坏原页面，单段失败不影响其他段落。
7. 网络检查确认核心翻译过程没有向远程翻译服务发送正文。
8. 用户可以清除设置和翻译缓存。
9. Chrome 重载扩展、刷新页面和单页应用导航均不会产生失控任务。
10. 自动化测试覆盖文本分段、去重、缓存键、任务取消和安全渲染。

## 9. 后续候选能力

- 视频字幕双语显示。
- 普通文本 PDF 的结构化翻译。
- 图片 OCR 后翻译。
- 自定义术语库导入和导出。
- 翻译质量反馈与本地偏好学习。
- 本地上下文润色、段落摘要和术语解释。
- 独立桌面端及跨浏览器实现。
